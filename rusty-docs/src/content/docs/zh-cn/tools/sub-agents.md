title: 子代理
description: 为复杂任务派生独立的子代理
---


## 概述

`agent` 工具派生一个子代理来独立处理复杂的子任务。子代理在自己的上下文中运行，拥有独立的消息历史，非常适合委派研究、并行探索或多步工作。

## 参数

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `task` | string | 是 | 给子代理的任务描述 |
| `context` | string | 否 | 额外的上下文或约束 |

## 子代理如何运作

1. 父代理带上任务描述调用 `agent` 工具。
2. 一个新的 `Agent` 实例在独立的 Tokio 任务中派生。
3. 子代理拥有自己的消息历史并独立运行。
4. 子代理可访问除 `agent` 工具本身以外的所有工具（防止递归派生）。
5. 子代理以 `BypassPermissions` 模式运行，因此不会请求权限。
6. 子代理完成后，其最终响应返回给父代理。

## 使用场景

**研究任务：** 委派需要多步调查的研究问题。

```json
{
  "task": "找出整个代码库中 `RustyError` 枚举的所有使用处，并按变体分类。",
  "context": "聚焦 agent 与 tools crate 中的错误处理模式。"
}
```

**并行探索：** 派生多个子代理，同时探索代码库的不同部分。

```json
{
  "task": "解释 provider crate 中的流式实现。",
  "context": "查看 openai.rs 与 types.rs。聚焦 SSE 解析与事件累积。"
}
```

**隔离分析：** 运行不应污染主对话上下文的分析。

```json
{
  "task": "分析 tools crate 的测试覆盖率，找出未被测试的边界情况。"
}
```

## 约束

- 子代理不能再派生子代理（无递归嵌套）。
- 子代理绕过所有权限提示。
- 子代理与父代理共享同一工作目录与文件沙箱。
- 子代理结果以单个文本响应返回；中间步骤对父代理不可见。
