Why Rusty

The macOS WindowServer Bottleneck

Web-tech developer tools offload rendering to local Electron or Chrome layers, forcing macOS WindowServer to continuously recompute compositing arrays. Opening two Node/TS agent sessions can consume 50% of system CPU purely to draw the interface.

Rusty writes raw terminal character arrays directly to the active shell viewport via hardware-supported ioctl buffers. WindowServer rendering overhead drops by 99%, leaving CPU power available for actual compilation and model operations.

Privacy Guarantee

Zero analytics, middleware, or backdoors.

Rusty does not phone home to any telemetry server. No trackers, no instrumentation libraries, no telemetry toggles. Requests stream strictly between your terminal and the LLM endpoint you configure.

Capabilities

Built for complex, multi-step work

Rusty coordinates multiple tools, manages task plans, and adapts its reasoning budget dynamically to context pressure.

Auto-Plan with Task Tracking

Launch with --plan-with-tasks and the agent creates a structured task list before executing. Tasks move through pending, in-progress, and completed states. Plan mode (--permissions plan) restricts to read-only for safe exploration.

Concurrent Tool Execution

Multiple independent tool calls in a single response execute in parallel via tokio::JoinSet. Results collected in call-order. Reduces wall-clock time for multi-file edits and batch searches.

Multimodal Image Support

Paste images directly into the TUI. Screenshots are sanitised (ANSI, control characters, bidi overrides stripped), base64-encoded, and sent as ContentBlock::Image to vision-capable models. Capped at 100KB per paste.

Model Registry & Picker

Define multiple models in settings.json with per-model API keys, endpoints, and context windows. Switch mid-session via /settings or --model from the CLI.

Dynamic Thinking Budget

Three levels (minimal/normal/deep) control reasoning token allocation. Multi-step tasks auto-boost to normal. High context usage (>70%) triggers step-downs to preserve output space. Set via --thinking-budget or /settings.

Per-Project Memory

Store project-specific notes and decisions in ~/.rusty/memory/. Save, search, list, and delete. Capped at 100 entries per project with FIFO eviction and multi-layer content sanitisation.

Multi-Provider Support

OpenAI-compatible and native Anthropic wire formats. Built-in presets for Xiaomi MiMo, Kimi, DeepSeek, OpenAI, and Ollama. Per-model API keys, endpoints, and context windows. The setup wizard handles provider configuration on first run.

Session Persistence

Full message history saved to ~/.rusty/sessions/ as JSON. Resume with --resume or /resume. Auto-compaction summarises older messages at ~80k tokens.

Operational Logic

Hardened local boundaries

Path Range Isolation

Every file write, read, and edit is verified via resolve_path(), which canonicalises and validates paths stay within the working directory. TOCTOU-hardened: pre-write and post-write symlink escape checks prevent race conditions.

Command Classification

Bash commands are classified per-invocation. Read-only commands (git status, cargo check) bypass write permissions. Destructive commands require explicit approval.

SSRF Protection

The web_fetch tool blocks localhost, private IPs, link-local addresses, and DNS rebinding. Every redirect is re-validated. IP pinning prevents hostname resolution bypasses.

Protected Files

The agent refuses writes to sensitive paths including .git/config, Cargo.lock, .env, and other protected files. Configurable via settings.json.

Tiered Permission System

Four modes: Default (interactive approval), AcceptEdits (auto-approve file writes), Bypass (allow all), Plan (read-only). Permanent allowlists persist across sessions in settings.json.

File Permission Hardening

Config directories (~/.rusty/) created with 0o700 permissions. Settings and memory files created with 0o600 permissions.

Open Copyleft Licensing

Every module operates under AGPL-3.0-or-later. No proprietary tracking middleware, background phone-homes, or telemetry is permitted in the compilation stream.

Configuration

Settings, credentials, and presets

Rusty resolves configuration through a layered chain: environment variables take highest precedence, followed by CLI flags, then the settings file. The setup wizard populates ~/.rusty/settings.json on first run.

Settings File (~/.rusty/settings.json)

{
  "api_key": "sk-...",
  "api_base": "https://api.xiaomimimo.com/v1",
  "default_model": "mimo-v2.5-pro",
  "models": [
    {
      "group": "Xiaomi",
      "name": "MiMo Pro",
      "provider": "OpenAI",
      "api_base": "https://api.xiaomimimo.com/v1",
      "model": "mimo-v2.5-pro",
      "available_models": ["mimo-v2.5-pro", "mimo-v2-flash"],
      "context_window": 131072,
      "thinking_budget": 4096
    }
  ],
  "active_model": "mimo-v2.5-pro",
  "allowed_tools": [
    "bash:git status",
    "bash:cargo check"
  ],
  "credential_store": "keyring",
  "thinking_level": "normal",
  "permission_mode": "default"
}

The allowed_tools array is populated by permanent permission grants during the session. Once you approve a tool and choose "always allow", it is written here and persists across sessions.

API Key Resolution Chain

Credentials are resolved in this priority order. The first non-empty value found is used:

1
RUSTY_API_KEY
Environment variable. Highest priority. Empty strings treated as absent.
2
OPENAI_API_KEY
Environment variable. Standard OpenAI-compatible key. Empty strings treated as absent.
3
OS Keyring
macOS Keychain, Linux Secret Service, or Windows Credential Manager. Used when credential_store is set to keyring.
4
Settings File
The api_key field in ~/.rusty/settings.json. Fallback when no other source is available.

Sessions

Every conversation is automatically saved to ~/.rusty/sessions/ as JSON with full message history, model, and timestamps. Sessions are isolated per-project.

Auto-save: Sessions save on /quit, /exit, or Ctrl+C.
Resume: --resume SESSION_ID or /resume in TUI.
List & Browse: --list-sessions in CLI or /sessions in TUI for interactive picker.
Rename: Organise sessions with /rename for easy identification later.
Compaction: Progressive micro-compaction prunes tool results and older messages based on token pressure, keeping recent context intact. Summarisation kicks in at ~80k tokens or 40 messages. Manual: /compact.
